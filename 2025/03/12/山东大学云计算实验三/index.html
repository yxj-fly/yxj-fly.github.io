云计算实验三

学号与个人信息

  • 学号：202200130012
  • 姓名：杨晓娟
  • 班级：22级2班

实验题目与基本信息

  • 实验题目：利用云平台搭建个人博客
  • 实验学时：2
  • 实验日期：2025年3月12日

实验目的

本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客/网站上。

硬件环境与软件环境

硬件环境

  • 联网的计算机一台

软件环境

  • Windows 或 Linux

实验步骤与内容

可以通过访问网站https://yxj-fly.github.io/来访问我创建的博客。

一、准备工具

为还说呢么图片会错误。

  1. 首先需要根据官网安装Git和Node.js，然后在终端检查是否安装成功。
  2. 然后在Github官网注册一个账号。
  3. 之后安装Hexo。在D盘下新建一个文件夹Myblog用来存放博客文件，在此文件夹下右键打开Git bash，输入npm install -g hexo-cli进行安装，完成后输入hexo -v检查是否安装成功。

二、建设网站

  1. 在Git bash中使用hexo init初始化网站，成功后Myblog文件夹下出现多个文件和文件夹。

  2. 在bash输入hexo g生成静态网页。
  3. 输入hexo s打开本地服务器，浏览器中输入给出的网址http://localhost:4000/浏览博客主页，Ctrl+c可以关闭本地服务。

  4. 在bash中输入hexo new post "云计算实验三"来发布新博客，此时的博客只能本地访问。

三、连接本地与远程服务器

  1. 新建Github仓库，仓库名为：yxj-fly.github.io。注意一定要使用用户名+github.io这种格式才能分配到http://username.github.io这样的域名。
  2. 生成SSH密钥，在bash中输入git config --global user.name "yxj-fly"和git config --global user.email "202200130012@mail.sdu.edu.cn"。注意用户名和邮箱要用注册Github的用户名和邮箱。可以输入git config --list来检查用户名和邮箱是否正确。
  3. 然后输入ssh-keygen -t rsa -C "202200130012@mail.sdu.edu.cn"来生成密钥，全部选项都用回车即可。
  4. 在文件夹C:\用户\DELL\.ssh下找到id_rsa.pub文件，复制文件里的全部内容。
  5. 在Github中找到settings，再找到SSH and GPG keys，新建一个密钥，粘贴刚才复制的内容。
  6. 在git bash输入ssh -T git@github.com，有选项就输入yes，成功后如下图。

四、将项目部署到Github

  1. 修改Myblog下的_config.yml文件，补充部署到Github的相关配置。
  2. 在bash下输入npm install hexo-deployer-git --save安装工具，然后使用hexo d将项目推送到Github，打开仓库看到项目推送成功。

  3. 在仓库中打开Settings，找到Pages，正确的配置如图。
  4. 完成之后打开Github给出的网址。
  5. 但是此时博客内没有内容，需要对D:\Myblog\source\_posts中的内容进行编辑，然后保存。
  6. 之后在git bash中输入hexo g和hexo d来重新生成文件并上传至github，进入网页后可看到更新后的内容。

五、报错分析

  1. 首先我遇到了hexo new post来发布新博客不能正常修改博客的问题，原因是双引号中英文格式弄错了，这让我更加的细心。
  2. 在尝试使用Hexo部署到GitHub Pages时，错误信息提示无法连接到GitHub的服务器（github.com）在端口443上。

    经过研究发现，虽然已经成功生成了SSH密钥并能够通过SSH认证GitHub，但错误信息显示Hexo尝试通过HTTPS而不是SSH来推送代码。所以需要打开Hexo博客目录下的 _config.yml 文件，检查 deploy 部分，使用 SSH 而不是 HTTPS。下图为修改前后：

  3. 此外遇到了进行hexo g和hexo d来重新生成文件并上传至github时，必须进行两次才能正确修改博客的问题。下图为第一次进行hexo g和hexo d的结果，但该问题不影响正常使用。
  4. github连接缓慢可尝试Windows系统打开cmd，输入命令得到github的ip地址，找到networks文件在最下面的空行粘贴ip地址，并加上github域名，即20.205.243.166 github.com，可缓解该问题。

结论分析与体会

通过本次实验，我成功掌握了基于云计算环境的网站部署技能，包括Git、Node.js、Hexo等工具的使用，以及如何将个人博客部署到Github Pages上。在实验过程中，我遇到了一些问题，如双引号格式错误、Hexo通过HTTPS推送代码等，但通过查阅资料和细心排查，最终成功解决了这些问题。

此外，我还深刻体会到了云计算技术在现代网站部署中的重要性。通过云平台，我们可以轻松地搭建和管理个人博客或网站，无需担心服务器硬件和维护成本。同时，云平台还提供了丰富的功能和工具，帮助我们更高效地开发和部署网站。

最后，我将本次实验报告发布在自己的博客上，作为对实验成果的总结和展示。这不仅是对自己学习成果的肯定，也是对未来学习和工作的一种激励和鞭策。
